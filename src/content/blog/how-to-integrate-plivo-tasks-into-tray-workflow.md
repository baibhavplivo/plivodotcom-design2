<p>Tray is an application integration platform that lets users connect applications to create complex workflows. It <a href="https://tray.io/documentation/connectors/service/plivo/">supports Plivo</a>, so you can create workflows that trigger Plivo to send an SMS or MMS message or make a voice call and speak text.</p><p>To start using Plivo with Tray, you need a <a href="https://tray.io/lp/get/get-started">Tray account</a> — sign up for a free trial — and a voice- or SMS-enabled Plivo phone number. You can purchase numbers from the <a href="https://console.plivo.com/active-phone-numbers/">Phone Numbers</a> page of the Plivo console, or by using the <a href="https://www.plivo.com/docs/numbers/">Phone Number API</a>.</p><figure style="max-width:1440px" data-rt-max-width="1440px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657ff0bc84117f80c254fa4f_buy-new-number.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h2>A sample Plivo/Tray integration</h2><p>To see how Tray works, we’ll create a workflow that integrates Plivo and Salesforce. In this workflow, every time a Salesforce Opportunity record is updated, a trigger gets fired. If the change that fired the trigger was to update the stage name to “Closed Won,” then Tray will call Plivo to send an MMS message with the opportunity details.</p><figure style="max-width:2870px" data-rt-max-width="2870px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657ff0de5b17e17950e3be21_complete-flow.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>To get started, open Tray’s Workflow Editor and click <strong>New workflow</strong>. Give your workflow a name (we used Salesforce Opp Trigger), then click <strong>Next</strong>. On the next screen you can choose how the integration gets triggered. Tray offers eight options, or you can choose an external application. That’s what we need — choose Salesforce, then click <strong>Create workflow</strong>.</p><ul><li>Click on the Salesforce trigger to configure it. At the top of the properties panel at the right of the screen select “On Record Update” as the action.</li><li>Click <strong>Describe</strong> and rename the tool Salesforce Opp Trigger.</li><li>Click <strong>Authenticate</strong> and enter your Salesforce credentials.</li><li>Click <strong>Input data</strong>, and from the Record type drop-down, select “Opportunity.”</li></ul><p>With that, we’ve told Tray to trigger the workflow every time a Salesforce Opportunity record is updated.</p><p>When the workflow is triggered, we want to check whether the new value of the Stage field is “Closed Won,” and if it is, we want to take another action — namely, send a text message.</p><p>To check the value, click the plus sign under the connector to add another Salesforce service connector, but this time set the action at the top of the properties panel to “Find records.”</p><ul><li>In the properties panel, click <strong>Describe</strong> and rename the tool Get Opportunity Details.</li><li>Click <strong>Authenticate</strong> and enter your Salesforce credentials.</li><li>Click <strong>Input Data</strong>, and from the Record type drop-down, select “Opportunity.”</li><li>Under Fields, click <strong>Add to Fields</strong> and add the Name, Account ID, Stage, Amount, and Close Date fields. We’re retrieving more than just the Stage field because we plan to use the values of the other fields in our future text message.</li><li>Now we specify that we want these field values only from the record with the Opportunity ID that triggered the workflow. Under Conditions, click <strong>Add to Conditions</strong>. A new condition will appear. Set Field to Opportunity ID and Operator to Equal to. For Value, click on the datatype dropdown and choose jsonpath. Then click in the field to pick the ID, which then shows up in the box as $.steps.trigger.events[0].Id. Alternatively, you can drag the mouse from the Value setting to the Salesforce Opp Trigger icon and select the ID field there.</li></ul><video autoplay="" loop="" muted="" inline="" width="560" height="315">

</video><p>Now we can check whether the Stage is Closed Won.</p><p>Click the plus sign again to add another connector at the bottom, and this time choose Boolean under Logic Tools. Click <strong>Boolean Condition</strong>.</p><ul><li>In the properties panel, click <strong>Describe</strong> and rename the tool Check If Closed Won.</li><li>Click <strong>Input data</strong> and fill in some required values.</li><li>For 1st Value field, click the datatype drop-down and choose jsonpath. Then click in the field and choose Stage, which will show up as $.steps.salesforce-1.records[0].StageName. Alternatively, you can drag the mouse from the 1st Value setting to the previous step in the workflow and select the stage name field there.</li><li>For Comparison type, use Contains.</li><li>For 2nd Value, enter “Closed Won” (without the quotation marks).</li></ul><video autoplay="" loop="" muted="" inline="" width="560" height="315">
 &nbsp;<source width="560" height="315" src="https://www.plivo.com/assets/posts/images/blog/tray/step3.mp4" type="video/mp4">
</video><p>If the boolean test returns false, meaning it’s not a Closed Won, we don’t have to do anything. But if the test returns true, we have to retrieve the account name from the Account record so we can include it with the other information we already retrieved in our text message.</p><p>Click the empty box in the True branch and add another Salesforce service connector.</p><ul><li>In the properties panel, click <strong>Describe</strong> and rename the tool Get Account Details.</li><li>Set the action at the top of the configuration pane to “Find records.”</li><li>Set Record type to “Account.”</li><li>Under Fields, click <strong>Add to Fields</strong> and add the Account Name field.</li><li>Under Conditions, click <strong>Add to Conditions</strong>. A new condition will appear. Set Field to Account ID, Operator to Equal to, and Value to $.steps.salesforce-1.records[0].AccountId. Alternatively, you can drag the mouse from the Value setting to the previous step in the workflow and select the stage name field there. This gives us access to the Account Name of the Account record whose ID matches the Account ID we retrieved earlier, which we’ll use in the text message we plan to send.</li></ul><video autoplay="" loop="" muted="" inline="" width="560" height="315">
 &nbsp;<source width="560" height="315" src="https://www.plivo.com/assets/posts/images/blog/tray/step4.mp4" type="video/mp4">
</video><p>Now we have all the information we need to send a text message. Click the plus sign under the previous connector and choose the Plivo service connector.<br></p><ul><li>In the properties panel, click <strong>Describe</strong> and rename the tool Celebrate Using Plivo.</li><li>Set the action at the top of the properties panel to “Send an MMS.”</li><li>Click <strong>Authenticate</strong>, then enter your Plivo authentication information. You can find your Plivo Auth ID and Auth Token on the overview page of the <a href="https://console.plivo.com/dashboard/">Plivo console</a>.</li><li>Click Input data. For Source, enter the number you want to be the caller ID for the text message. Set Destination to be the phone number you want to receive the text message. Set Text to: New opportunity closed won, let’s celebrate! Account Name: {$.steps.salesforce-2.records[0].Name} Owner Name: {$.steps.salesforce-1.records[0].Opportunity_Owner_Name__c} Closed Date: {$.steps.salesforce-1.records[0].CloseDate} To enter the variable names, begin typing {$ and you’ll see a list of values from which to select. Click Add to Media URLs. Set Media URL to “” which is the address of a happy GIF file. If you left this off, you could send an SMS message instead of <a href="/blog/what-is-mms-messaging/">MMS</a>; the process is otherwise the same.</li></ul><video autoplay="" loop="" muted="" inline="" width="560" height="315">

</video><p>Your workflow is now complete. <strong>Click Enable workflow</strong> at the bottom of the page, then test your configuration by marking an opportunity in a Salesforce Opportunity record Closed Won and checking the device associated with the destination phone number for the message you’re expecting.</p><p><strong>Note:</strong> If you’re using a Plivo Trial account, you can make calls only to phone numbers that have been verified with Plivo. You can verify (sandbox) a number by going to the console’s Phone Numbers <a href="https://console.plivo.com/phone-numbers/sandbox-numbers/">Sandbox Numbers</a> page.</p><p>Haven’t tried Plivo yet?
