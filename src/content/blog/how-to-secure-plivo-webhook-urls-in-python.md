<p>Plivo customers control call flows and business logic based on webhook requests Plivo sends to their servers, which also convey critical information such as call records and message status. To keep these webhooks secure, we enable signature validation using hash-based message authentication codes (<a href="https://en.wikipedia.org/wiki/HMAC">HMAC-SHA256</a>) so you can check whether the webhook delivered to your server originated from Plivo. This post walks you through how to secure webhooks and authenticate webhook requests using the Plivo Python SDK.</p><p>All voice API requests from Plivo’s platform to your application server contain three custom HTTP headers for <a href="https://www.plivo.com/docs/voice/concepts/signature-validation/">signature validation</a>: X-Plivo-Signature-V3, X-Plivo-Signature-Ma-V3, and X-Plivo-Signature-V3-Nonce. (For SMS we use X-Plivo-Signature-V2 as of now.) To verify that the request to your server originated from Plivo and that it was not altered en route, you can generate your own request signatures and compare them with the Plivo-Signature headers we pass.</p><h3>Validating requests on the application server</h3><p>Plivo Server SDKs include functions to help you validate incoming requests with X-Plivo-Signature HTTP headers. Here’s some sample code that shows how to validate a signature and return an XML element if the webhook is authenticated.</p><p>To start, create a working directory and change into it. Within the directory, create a Python virtual environment and activate it, then install the Python libraries we’ll be using.</p><p>Create a file named signature.py and paste into it this code.</p><p>When someone makes an outbound call to a destination number, Plivo requests the webhook associated with this code. The code runs and returns a Speak XML element if it can authenticate the request sent. If the validation fails the call will terminate, since no XML is returned to the Plivo server.</p><p>Run the file using the command python signature.py to initialize the server locally.</p><p>When you’re satisfied, make the application publicly available with <a href="https://www.plivo.com/docs/sdk/server/set-up-python-dev-environment-api-xml-voice/#ngrok-setup">ngrok</a>. To test it, make a call to the API using <a href="https://www.plivo.com/docs/voice/quickstart/postman/">Postman</a> and use the ngrok URL as the answer URL.</p><figure style="max-width:2122px" data-rt-max-width="2122px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d9b758331325e70d478c6_postman.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h3>Stay secure</h3><p>Be sure to verify your webhooks using Plivo’s Python SDK and avoid causing any harm to your servers. You can learn more about signature validation logics for <a href="https://www.plivo.com/docs/voice/concepts/signature-validation/">voice</a> and <a href="https://www.plivo.com/docs/sms/concepts/signature-validation/">messaging</a>.</p><p>By the way, signature validation isn’t the only technique we use to keep webhooks secure. All webhook communications run over HTTPS, which are secured by SSL certificates on your server. We also provide a list of IP addresses for our voice API from which you can expect callbacks. You can <a href="https://www.plivo.com/docs/voice/concepts/ip-address-whitelisting/#callbacks-from-plivo">whitelist</a> these addresses in your infrastructure.</p><p>Haven’t tried <a href="https://www.plivo.com/">Plivo</a> yet?
