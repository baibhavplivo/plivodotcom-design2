<p>When clients and prospects ring your phone number, you need to have someone answering their calls. That “someone” could be an auto-attendant phone system — a prerecorded or computer-generated voice that serves as your business’s virtual receptionist. Virtual receptionists are cost-effective, provide a professional impression to customers, and are easy to integrate.</p><p>I looked at the most popular business phone systems that include an auto-attendant feature. I found no consistency in terms of the features they offer or the price, other than the more you pay, the more features you get — and if you choose a solution, you’re contractually bound to them and get only the features they offer.</p><p>My research inspired me to attempt and build a virtual receptionist using <a href="https://www.plivo.com/phlo/">PHLO</a>, Plivo’s visual workflow design studio. I created a call flow based on interactive voice response (IVR) and extension-based call forwarding to meet my needs, and if I need to customize it further, I can do that.</p><h2>Basic IVR</h2><p>In my setup, basic IVR greets callers with a welcome message and offers choices for the caller. We’ve written a <a href="https://www.plivo.com/docs/voice/use-cases/ivr/node/">getting started guide to using PHLO for IVR</a>, which I used as a springing-off point — look to that for the step-by-step setup process.</p><p>I entered the text that explains to callers key to press to reach our sales team and which for support. Alternately, you can upload an MP3 file that does the same thing. I connected the IVR menu choices to the next logical action — I dragged Call Forward components to the canvas and connected to them.</p><figure style="max-width:616px" data-rt-max-width="616px"><div><img src="/images/blog/6582f3de6ef34a9b86604fdc_image04.png" alt="IVR Menu 4" width="auto" height="auto" loading="auto"></div></figure><h2>Extension-based call forwarding</h2><p>To implement extension-based call forwarding, I used a third number from the IVR Menu component with a voice prompt to tell callers, “If you know your party’s extension, please press 3.” If the caller does that, we then need to prompt them to enter the extension, then capture the digits they enter. I dragged the Get Input component to the canvas and set up a voice prompt on it.</p><figure style="max-width:668px" data-rt-max-width="668px"><div><img src="/images/blog/6582f3de5adb54e1c946784b_image05.png" alt="IVR Menu 5" width="auto" height="auto" loading="auto"></div></figure><p>Get Input captures entered digits and passes on to the next node — in this case a Branch component that compares the input and sets up the next action…</p><figure style="max-width:668px" data-rt-max-width="668px"><div><img src="/images/blog/6582f3de4141ab747d419231_image06.png" alt="IVR Menu 6" width="auto" height="auto" loading="auto"></div></figure><p>… which here is a Play Audio component prompt to inform the caller where the call is getting forwarded to, followed by a Call Forward component.</p><figure style="max-width:442px" data-rt-max-width="442px"><div><img src="/images/blog/6582f3def7273ce87907de5a_image07.png" alt="IVR Menu 7" width="auto" height="auto" loading="auto"></div></figure><p>I named the PHLO, validated it, and saved it. Finally, following the steps in the guide, I attached the PHLO to a phone number from my Plivo account, and my virtual receptionist was ready to go.</p><p>Our auto-attendant worked great — and that was just the start! I built many other features on top of this basic setup, including SMS acknowledgement for callers, voicemail for callback, and time-based routing.</p><p>To try PHLO yourself, <a href="https://console.plivo.com/accounts/register/">sign up</a> for a free trial account.</p>
