<p>Looking for support for Auth0 authentication? Here are the steps to take to add SMS-based multifactor authentication (MFA) to the login flow for an Auth0 tenant. (A tenant is a specification for a group of users who share access to an application instance — for instance, a company with multiple employees.) We’ll assume you already have an Auth0 account and tenant and a <a href="https://console.plivo.com/accounts/login/">Plivo account</a>.</p><p>You must have a voice-enabled Plivo phone number to send SMS messages to numbers in the US and Canada; you can rent numbers from the<a href="https://console.plivo.com/active-phone-numbers/">Numbers</a> page of the Plivo console, or by using the <a href="https://www.plivo.com/docs/numbers/">Numbers API</a>.</p><p>We recommend testing your setup on a <a href="https://auth0.com/docs/dev-lifecycle/set-up-multiple-environments">staging or development server</a> before making changes to your production login flow.</p><h2>Add the Action</h2><p>To integrate Auth0, first <a href="https://auth0.com/signup">sign up</a> for an account. Add an <a href="https://auth0.com/docs/actions">Action</a> (a triggerable function), then integrate it with your authentication flow.</p><ul><li>Go to Actions &gt; Library and select Add Integration. </li></ul><figure><div><img src="/images/blog/658030f439f61b597ac319ca_actions-library.png" alt="actions-library" width="auto" height="auto" loading="auto"></div></figure><ul><li>Read the necessary access requirements and click <strong>Continue</strong>. </li></ul><figure><div><img src="/images/blog/658030f4b29d515ac2dcf1e9_requirements.png" alt="requirements" width="auto" height="auto" loading="auto"></div></figure><ul><li>Configure the integration by filling in the fields on the next screen with your Plivo Auth ID and Auth Token and your Plivo phone number. </li></ul><figure><div><img src="/images/blog/658030f47367791cbdeb28e5_configuration.png" alt="configuration" width="auto" height="auto" loading="auto"></div></figure><ul><li>Click <strong>Create</strong> to add the integration to your library.</li><li>Click the <strong>Add to flow</strong> link on the pop-up that appears. </li></ul><figure><div><img src="/images/blog/658030f4145e6b58c98c81bd_add-flow.png" alt="add-flow" width="auto" height="auto" loading="auto"></div></figure><ul><li>Drag the Action into the flow. </li></ul><figure><div><img src="/images/blog/658030f49cf2ecccade988aa_send-sms.png" alt="send-sms" width="auto" height="auto" loading="auto"></div></figure><ul><li>Click Apply Changes. Now this flow will use the Plivo integration to send an SMS message whenever it’s called.</li></ul><h2>Activate custom SMS factor</h2><p>Before you can use SMS as an authentication factor, your Auth0 tenant needs to have <a href="https://auth0.com/docs/login/mfa/enable-mfa">MFA enabled globally</a> or <a href="https://auth0.com/docs/login/mfa/customize-mfa-user-pages">for specific contexts</a>. You can then configure the SMS factor to use your custom code.</p><p>Go to <a href="https://manage.auth0.com/select-tenant?path=/mfa">Dashboard &gt; Security &gt; Multi-factor Auth</a> and click the Phone Message factor box. In the modal that appears, select Custom for the delivery provider, make any adjustments you’d like to the templates, then click Save and close the modal. Finally, enable the SMS factor using the toggle switch.</p><figure><div><img src="/images/blog/658030f440896ac0d9ab1ee6_sms-factor.png" alt="sms-factor" width="auto" height="auto" loading="auto"></div></figure><p>Auth0 will immediately begin using this factor for MFA during login. Before you activate your integration in production, make sure you’ve configured all of the components correctly and verified everything on a <a href="https://auth0.com/docs/dev-lifecycle/set-up-multiple-environments">test tenant</a>.</p><h2>Test MFA flow</h2><p>Navigate to the Authentication section in the Auth0 Manage Dashboard, choose your Connection, then select <strong>Try</strong> from the connection’s dropdown menu to verify that everything works as intended.</p><figure><div><img src="/images/blog/658030f454765f5aad81650f_db-connection.png" alt="db-connection" width="auto" height="auto" loading="auto"></div></figure><p>You can then log into your Plivo account to verify that SMS messages are indeed being sent.</p><figure><div><img src="/images/blog/658030f474351fdd7e47711a_plivo-logs.png" alt="plivo-logs" width="auto" height="auto" loading="auto"></div></figure><p>If you don’t receive an SMS message as expected, look in your <a href="https://auth0.com/docs/logs">tenant logs</a> for a failed Phone Message log entry. To learn which event types to search, see the <a href="https://auth0.com/docs/logs/log-event-type-codes">Log Event Type Code list</a>. You can use the Filter control to find MFA errors.</p><p>Make sure that:</p><ul><li>The Action is in the Send Phone Message flow.</li><li>The secrets are the same Plivo Auth ID and Auth Token you created when you added the Action.</li><li>Your Plivo account is active (not suspended).</li><li>Your phone number is formatted in <a href="https://en.wikipedia.org/wiki/E.164">E.164 format</a>.<br></li></ul><p></p>
