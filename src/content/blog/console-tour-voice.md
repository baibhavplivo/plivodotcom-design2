<p>We began our tour last time with a look at <a href="http://plivo-webflow.webflow.io/blog/console-tour-sms">what you can do on the Plivo console to monitor and manage our SMS API</a>. The series continues here with a similar survey for our <a href="https://plivo-webflow.webflow.io/voice">Voice API</a>.</p><h2>The Voice menu overview</h2><p>To get to the Voice menu, visit the <a href="https://console.plivo.com/dashboard/">Plivo console</a> and click on the second icon.</p><figure><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d508f45a3814e21592284_vt01.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>The console displays the Voice Overview page, which shows some basic usage information.</p><figure style="max-width:2114px" data-rt-max-width="2114px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d50b9ac1b24462954fe68_vt02.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h2>Advanced Insights</h2><p>Moving to the next menu item takes you to the <a href="https://plivo-webflow.webflow.io/voice/call-insights">Advanced Insights</a> page. It shows information that can help you identify call quality issues so you can fix them quickly.</p><figure style="max-width:2154px" data-rt-max-width="2154px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d50e5e263ba3262e99c70_vt03.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h2>XML and PHLO applications</h2><p>The next menu choice brings you to a list of the Plivo XML applications and <a href="https://www.plivo.com/docs/phlo/">PHLOs</a> associated with your account. &nbsp;In Plivo parlance, an application is a set of answer, hangup, and message URLs that you use to control what happens to your incoming calls and messages, while our PHLO visual workflow design tool lets you build applications graphically. Clicking on an application name brings you to a screen of details about the application.</p><figure style="max-width:2168px" data-rt-max-width="2168px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d510989d3663accd847ab_vt04.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>The entries in the Voice section of this page define URLs the application looks at in the event of an incoming call to that number or when the call is hung up. Upon receipt of an incoming call Plivo makes a synchronous HTTP request to your application using the URL specified in the Primary Answer URL field. The API expects an XML document in response. Plivo also sends a few parameters with the HTTP request that your application can act upon before responding. </p><p>If there were a URL specified for messaging it would work the same way. You can read more about the process in our <a href="https://www.plivo.com/docs/sms/xml/overview/">documentation</a>.</p><h2>Endpoints</h2><p>The next menu choice is Endpoints, which displays a list of <a href="https://www.plivo.com/docs/voice/concepts/sip-endpoint/">SIP endpoints</a> associated with your account. A SIP endpoint can be any IP phone, mobile phone, wireless device, or PC that uses Session Initiation Protocol (SIP) to perform communications operations. </p><h2>Logs</h2><p>Click on the Logs submenu to bring up the Voice log: a list of every voice API URL call made, which you can filter a number of ways or export for analysis elsewhere.</p><figure><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d513d74351fdd7eac9d12_vt05.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>When you click on a call entry on the log page, you can see details about the call.</p><figure style="max-width:2300px" data-rt-max-width="2300px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d516762d48bffd491aa76_vt06.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>That’s a ton of information about the call itself — and if that doesn’t tell you enough, well, there’s more. A note in the last box on the page directs you to the call debug log entries. The debug log contains details for all callbacks that Plivo sends.</p><figure style="max-width:2234px" data-rt-max-width="2234px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d522b704ec3bbd1156a32_vt07.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>With all the information in the logs you should be able to understand any issues you encounter. (But if you need help, our support team is always here for you.)</p><p>In addition to call logs, we give you access to recording and transcription logs.</p><figure style="max-width:2292px" data-rt-max-width="2292px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d5255d116f1873fc1bdd9_vt08.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>If you click on an item in the log, you can see details about that item.</p><figure><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d527de0747baa9b3738f7_vt09.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h2>Settings</h2><p>The final group of menu items falls under the category of Settings. The first setting entry determines whether to use <a href="https://plivo-webflow.webflow.io/voice/call-insights">Call Insights</a>. Because the feature is still in beta and therefore free to everyone, it’s enabled by default.</p><p>The Push Credentials screen lets you store the credentials you need if you’re working with Plivo’s <a href="https://www.plivo.com/docs/sdk/client/ios/overview/">iOS SDK</a> or <a href="https://www.plivo.com/docs/sdk/client/android/overview/">Android SDK</a>. Once you have credentials issued by Apple and Google you can enter them here.</p><p>Geo Permissions can help you protect your account from fraud by restricting the set of countries your account can call. Get more information from our <a href="https://www.plivo.com/docs/voice/concepts/geo-permissions/">documentation</a>.</p><p>Finally, Other Settings lets you enable <a href="https://www.plivo.com/docs/sms/concepts/destination-number-validation/">destination number validation</a> (which can give you better delivery rates and cost savings from not sending messages to numbers that are incapable of accepting them), enable HTTP Basic Auth on all GET recording media requests for enhanced security, and providing consent for Plivo to store and have access to logs and media packet capture files.</p><h2>All about the console</h2><p>That’s our quick tour of the Voice menus. To dig deeper, visit <a href="https://console.plivo.com/dashboard/">your console page</a> and see what your data looks like. Also check out our other console tours of the <a href="http://plivo-webflow.webflow.io/blog/console-tour-sms">SMS</a>, <a href="http://plivo-webflow.webflow.io/blog/console-tour-zentrunk">Zentrunk</a>, <a href="http://plivo-webflow.webflow.io/blog/number-lookup-intro">Number Lookup</a>, and <a href="http://plivo-webflow.webflow.io/blog/console-tour-numbers">Phone Numbers</a> menus. If you have any questions, you can find the answers in our documentation or on our <a href="https://support.plivo.com/hc/en-us/categories/360001245191-Voice-API">support portal</a>.</p>
