<p>We’ve never taken our blog readers on a tour of the console. Did we assume that since they could sign up for free they could give themselves a self-guided tour? That’s certainly an option, but we’ll take a moment here and now to be your tour guide. Please keep together.</p><p>We’ll start our tour with a look at the SMS features available on the console. We were inspired to call out this section by a comment from Sharita Passariello, who works for Plivo customer Fluent, and who said in a recent customer story, “I love the self-service console and ease of use. It’s critical for us to check the SMS logs and see the messages that were delivered and undelivered, and see why messages were undelivered.”</p><p>Passariello shared additional insights about the state of <a href="https://www.plivo.com/sms/" target="_blank">SMS API</a> marketing in a recent webinar.</p><h2>The SMS menu overview</h2><p>To get to the SMS menu, visit the <a href="https://console.plivo.com/dashboard/" target="_blank">Plivo console</a> and click on the third icon.</p><figure><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d694dac1b244629632620_SMStour01.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>The console displays the Messaging Overview page.</p><figure><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d6989fc6fa186be1182eb_SMStour02.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>Below some account information, the overview page shows a quick usage summary, a map (holy Mercator, look at the size of Greenland!) with a table showing usage by location, and a graphical and tabular breakout of messaging by source number type.</p><h2>SMS and debug logs</h2><p>That’s all very pretty and good information to pass along to upper management, but the console exposes lots more information. To see it, click on the Logs submenu to bring up the SMS/MMS log: a list of every messaging API URL call made, which you can filter a number of ways or export for analysis elsewhere.</p><figure><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d69ae5bf5a83f4fa93c0e_SMStour03.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>When you click on a message entry on the log page, you can see details about the message.</p><figure style="max-width:2204px" data-rt-max-width="2204px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d69defb87095054106a6c_SMStour04_new.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>We won’t make this long post longer by pointing out the information in each section of the page — the point is, you can see all the information you need about a given message, including status, content, and charges.</p><p>In addition to the SMS/MMS log, for API calls that include callback URLs, we expose details of our API responses in a debug log; it contains details for all callbacks that Plivo sends. If you click on MessageUUID of an entry in the debug log you can zoom in and see all the details about the message.</p><figure style="max-width:510px" data-rt-max-width="510px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d6a0e3e44cc3168dfa0b7_SMStour08.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h2>10DLC</h2><p>Since Fluent’s Passariello also complimented us on our 10DLC console capabilities in the company’s customer story, let’s look at those next. From the 10DLC menu choice you can add the 10DLC brand you’ve registered for your business, or register one or more new ones. (Resellers, for instance, register multiple brands.) If you click on a brand, you can see all the campaigns registered for it, register new ones, and deactivate existing ones.</p><figure style="max-width:2396px" data-rt-max-width="2396px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d6a4874351fdd7ebb00c4_SMStour05.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>If you click on a campaign, you can see all of the phone numbers registered for it, and link and unlink numbers, as well as see the throughput allocation for the campaign.</p><figure style="max-width:2264px" data-rt-max-width="2264px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d6a6c89d3663acce82602_SMStour06.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><h2>XML and PHLO applications</h2><p>From the SMS menu you can also look at the Plivo applications associated with your account. In Plivo parlance, an application is a set of answer, hangup, and message URLs that you use to control what happens to your incoming calls and messages. This console page shows the mappings.</p><p>Here’s an example of an interactive voice response (<a href="https://www.plivo.com/docs/voice/use-cases/ivr/node/" target="_blank">IVR</a>) XML application; we can tell that by looking at the information about it in the Application Details section. The entries in the Voice section define URLs the application looks at in the event of an incoming call to that number or when the call is hung up. Upon receipt of an incoming call Plivo makes a synchronous HTTP request to your application using the URL specified in the Primary Answer URL field. The API expects an XML document in response. Plivo also sends a few parameters with the HTTP request that your application can act upon before responding.</p><p>Messaging works the same way. Plivo makes a synchronous HTTP request to your application using the URL specified in the Message URL field. Your web application should return an XML document that provides instructions to the Plivo API on how the event should be handled. You can read more about the process in our <a href="https://www.plivo.com/docs/sms/xml/overview/" target="_blank">documentation</a>.</p><figure style="max-width:1590px" data-rt-max-width="1590px"><div><img src="https://uploads-ssl.webflow.com/656ac26f3a3f6e1cc08ba7a5/657d6a9854838aafd21ede28_SMStour07.png" loading="lazy" width="auto" height="auto" alt=""></div></figure><p>In addition to XML applications, you can see information about <a href="https://www.plivo.com/docs/phlo/">PHLO</a> applications. Plivo High-Level Objects is a visual design studio that lets you build workflows with little or no code. When you click on a PHLO in the list, you bring up the design page for that PHLO, so you can change the workflow and settings.</p><h2>Sender IDs, MMS media upload, and Powerpack</h2><p>Many countries — but not the US or Canada — support <a href="https://www.plivo.com/docs/sms/concepts/sender-id-usage/">sender IDs</a>, which are like caller IDs but for messaging. From this menu page you can see what sender IDs you’ve registered and register new ones.</p><p>If you’re planning on sending <a href="https://www.plivo.com/blog/what-is-mms-messaging/">MMS messages</a> with image, audio, or video attachments, you can specify your own URL or you can use the MMS Media Upload screen to upload the multimedia files to Plivo. then use the media ID associated with the file in the SMS API to include the attachment. Hosting it with us gives you lower latency and eliminates the need for you to manage attachments elsewhere.</p><p>Plivo also offers <a href="https://www.plivo.com/sms/powerpack/">Powerpack</a>, which lets companies manage messaging at scale. With the introduction of <a href="https://www.plivo.com/blog/10dlc-10-digit-long-code/">10DLC</a>, which offers higher throughput on long code numbers than was formerly available, Plivo no longer recommends using Powerpack for US customers, but it’s a great tool for other countries.</p><h2>Settings</h2><p>The final SMS menu choice lets you tailor certain settings. For example, under Alerts, you can choose whether to notified when</p><ul><li>the delivery rate for outbound messages drops significantly</li><li>delivery failures for inbound messages rise significantly</li><li>SMS volumes display unusual surges</li></ul><p>Another screen lets you set <a href="https://www.plivo.com/docs/sms/concepts/geo-permissions/">geo permissions</a> as a way to help curb SMS fraud. You can select a list of destination countries to which outbound messaging from your account is allowed. We’ll block (and not charge for) any messages sent to countries not in this list. You can also define a list of countries to which outbound SMS traffic is allowed at a subaccount level.</p><p>The third and final settings submenu lets you tailor a few miscellaneous settings.</p><h2>All about the console</h2><p>The console is the nerve center for administering the Plivo platform in action. This post touched on some highlights; to dig deeper, visit your own console and see what your data looks like. Also check out our other console tours of the <a href="https://www.plivo.com/blog/console-tour-voice/">Voice</a>, <a href="https://www.plivo.com/blog/console-tour-zentrunk/">SIP Trunking</a>, <a href="https://www.plivo.com/blog/number-lookup-intro/">Number Lookup</a>, and <a href="https://www.plivo.com/blog/console-tour-numbers/">Phone Numbers</a> menus. If you have any questions, you can find the answers in our <a href="https://www.plivo.com/docs/sms/concepts/overview/">documentation</a> or on our <a href="https://support.plivo.com/hc/en-us/categories/360001247012-Messaging-API">support portal</a>.</p><figure style="padding-bottom:33.723653395784545%" data-rt-max-width="" data-rt-max-height="33.723653395784545%" data-rt-dimensions="854:480" data-page-url="https://youtu.be/CQp_zUx65D8"><div><iframe allowfullscreen="true" frameborder="0" scrolling="no" src="https://www.youtube.com/embed/CQp_zUx65D8" title="A Tour of the Plivo Console"></iframe></div></figure>
