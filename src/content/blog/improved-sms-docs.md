<p>We’ve had countless customers tell us that they love our product, but fewer people gush about our documentation. As an infrastructure company, having great docs is just as important as boosting call quality and SMS delivery rates. This is why, as part of our focus on making our product the best it can be, we’ve put together a small task force to improve our documentation. We hope that the improvements we’re making help ensure that both novice and seasoned developers have a great experience with Plivo.</p><p>We researched and referenced some of the documentation that we like, then prioritized the top six things we want to do:</p><ol><li><strong>Lower the learning curve</strong>: We often take our own knowledge for granted and forget how hard it was for each of us to learn a new platform. To lower that learning curve, we’re breaking down the Getting Started section into more foundational steps.</li><li><strong>Build a better foundation</strong>: By breaking down and simplifying each step, we’re trying build a better understanding for our developer customers. Each tutorial will build on the ones before it.</li><li><strong>More comprehensive</strong>: Telecom is full of technical jargon that’s important for conceptualizing how web and mobile apps interact with telecom infrastructure. Therefore, you’ll see us explain certain concepts in more detail to bring more transparency to the way telecom works.</li><li><strong>Focus on what’s important</strong>: To highlight what’s important, we’ve added note sections to mark up the essential parts to each tutorial.</li><li><strong>Better code markup</strong>: We’ve upgraded our syntax highlighter to include line numbers and support for more languages.</li><li><strong>More languages</strong>: Our Getting Started section is full of examples, but most of them were written in Python. You’ll now see how everything can be done in Ruby, Java, PHP, .NET, and Curl as well.</li></ol><p>To start things off, we’ve added more <a href="https://www.plivo.com/docs/sms/quickstart/php-server/">getting started tutorials for SMS</a>. And in the next few weeks, you can expect to see a lot more improvements.</p><p>Here are some of new SMS tutorials we’ve added:</p><ul><li><a href="https://www.plivo.com/docs/sms/use-cases/send-an-sms/node/">Send a Single SMS Message</a>: Send an outbound SMS text message to any phone number. Use cases: two-factor authentication, notifications, communication apps, etc.</li><li><a href="https://www.plivo.com/docs/sms/use-cases/receive-sms/node/">Receive an SMS Message</a>: Receive SMS text messages in 19 countries around the world. Use cases: ride hailing, delivery coordination, surveys and polling, two-factor authentication, etc.</li><li><a href="https://www.plivo.com/docs/sms/use-cases/reply-to-incoming-sms/node/">Reply to an Incoming SMS Message</a>: Interact with end users using SMS text messages. Use cases: product reviews, dating apps, purchase verification, etc.</li><li><a href="https://www.plivo.com/docs/sms/use-cases/forward-incoming-sms/node/">Forward an Incoming SMS Message</a>: Redirect incoming messages to another phone number. Use cases: help desks, sales teams, CRM, etc.</li><li><a href="https://www.plivo.com/docs/sms/use-cases/delivery-reports/node/">Handle SMS Delivery Reports</a>: Set up a web server to handle all of your delivery reports. Use cases: customer dashboards, analytics, calculating ROI, etc.</li><li><a href="https://www.plivo.com/docs/sms/use-cases/forward-sms-to-email/node/">Forward SMS to Email</a>: Forward SMS text messages on your phone number to your email inbox.</li></ul><p></p>
