<p>The <a href="https://www.plivo.com/docs/sdk/client/ios/overview/" target="_blank">Plivo iOS SDK</a> v2 has launched in public beta. iOS SDK helps you make and receive calls in your iOS applications. This SDK is compatible with devices running iOS version 8 and higher.</p><p>One major enhancement in Plivo iOS SDK v2 is that it supports <a href="https://developer.apple.com/documentation/pushkit" target="_blank">Pushkit</a> and <a href="https://developer.apple.com/documentation/callkit" target="_blank">Callkit</a>, meaning applications built using the SDK can receive incoming calls using Apple VoIP Push notifications. Pushkit also eliminates the need for persistent connections to receive incoming calls, thereby reducing device battery consumption by the application. Using Callkit, applications built on the new SDK can now handle VoIP calls with the same level of priority as that of a PSTN call.</p><p>To receive an incoming call using Pushkit, you just set up your push credentials, then handle push notifications using the Plivo SDK.</p><p>We’ve created a <a href="https://www.plivo.com/docs/sdk/client/concepts/mobile/setting-up-push-credentials/" target="_blank">guide to setting up push credentials</a> in the Plivo console. You can refer to our <a href="https://www.plivo.com/docs/sdk/client/ios/reference/" target="_blank">SDK Reference</a> for more information about APIs for handling the push notification. We’ve also developed a full-fledged <a href="https://github.com/plivo/plivo-ios-sdk2-examples" target="_blank">sample application in Swift</a> that implements all the features of the SDK.</p><p>You can get started with the SDK by installing it manually or by using Cocoapods. For manual installation, <a href="https://helpers.plivo.com/ios/plivo-ios-package-2.0.zip" target="_blank">download the SDK</a> and refer to our <a href="https://www.plivo.com/docs/sdk/client/ios/overview/" target="_blank">installation guide</a>. Alternatively, use our <a href="https://cocoapods.org/pods/PlivoVoiceKit" target="_blank">Cocoapods</a> to integrate your application with the SDK.<br></p><p></p>
